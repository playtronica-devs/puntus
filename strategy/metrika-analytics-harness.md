## 4. Срезы, которые тянем (и зачем)

| Срез (dimension) | Ищем ответ на вопрос | Действие при находке |
|------------------|----------------------|----------------------|
| Источник трафика (`lastTrafficSource`) | какой канал даёт деньги, а не просто визиты | перелив бюджета в конвертящий канал |
| Поисковая система (`lastSourceEngine`) | Яндекс vs Google — где покупают | SEO-приоритет |
| Устройство (`deviceCategory`) | мобайл vs десктоп CR (ювелирка = мобайл-тяжёлая) | чинить мобильную корзину, если CR ниже |
| Пол / возраст (`gender`/`ageInterval`) | кто реально платит; **сигнал по мужчинам (B1)** | таргет + инсайт для мужского бренда |
| Гео (`regionCity`) | города-доноры выручки | гео-таргет рекламы, логистика |
| Лендинг (`startURLPathFull`) | какие входные страницы конвертят/сливают | усилить конвертящие, чинить сливающие |
| День недели / час (`dayOfWeekName`/`hour`) | когда покупают (был пик в вс) | тайминг писем/постов/акций |
| Товары (`productName`) | что приносит выручку, что только смотрят | ассортимент, промо, допродажа |
| Поисковые фразы (`searchPhrase`) | язык спроса; **фразы про мужское/подарок (B1)** | SEO-контент, email-нарратив |
| Новый/вернувшийся (`isNewUser`) | CR новых vs вернувшихся | ретеншн-механики, ремаркетинг |

## 5. Аналитический слой (где «интересные выводы»)

После выгрузки — читаем и ищем **аномалии и рычаги**, не просто числа:
1. **Разрыв CR по срезу** — где конверсия резко ниже среднего (0.86%) при большом трафике = утечка денег.
2. **Разрыв «трафик vs выручка»** — канал/страница даёт много визитов, но мало заказов (или наоборот).
3. **Концентрация** — правило Парето: какие 20% товаров/городов/каналов дают 80% денег.
4. **Демография платящих** — кто в реальности платит vs кто ходит (особенно сигнал по мужчинам — B1).
5. **Тайминг** — когда деньги, чтобы туда ставить активности.

Выводы фиксируются в `strategy/metrika-insights-YYYY-MM-DD.md` (датированный отчёт) с приоритетом
по деньгам и оценкой усилия. RULE P1/P2: каждое число — с периодом и источником.
