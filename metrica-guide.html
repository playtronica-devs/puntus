🎯 Методология просмотра сессий Вебвизора

Протокол сбора UX-информации · v1.0
Принцип: Каждая сессия — это интервью без слов. Пользователь показывает где ему сложно, не говоря об этом. Задача — найти паттерны, а не единичные случаи. 3+ повтора = баг. 1 повтор = гипотеза.
📋 Шаг 1 — Отбор сессий для просмотра

🔥 Приоритет 1: Горячие лиды (>2 мин)

  • Время на сайте >2 мин — человек изучает
  • 3+ страниц — навигация активная
  • Возврат #2+ — лояльные
  • Источник: Instagram, прямой заход

🧊 Приоритет 2: Быстрые отказы (<10 сек)

  • Время 0:00–0:10 — чем триггирует уход?
  • С какой страницы зашли?
  • Откуда пришли? (UTM/соц.сеть)
  • Мобайл vs десктоп — есть ли паттерн?

🔄 Приоритет 3: Длинные без покупки

  • 5+ мин но 0 конверсий — где потерялись?
  • Открывали карточку товара?
  • Доходили до корзины?
  • Где курсор последний раз завис?

🌍 Приоритет 4: Диаспора (NL/DE/IL/CY)

  • Заходят с NL/DE/FR/GE = вероятно РФ+VPN
  • Пробуют оформить заказ? Где стопорятся?
  • Смотрят страницу доставки?
  • Язык интерфейса — читают кириллицу?

📋 Шаг 3 — Чек-лист по типам страниц

🏠 Главная / Коллекции (/spring, /rings)

  • Куда смотрит первый взгляд? (верх страницы)
  • Доскролливает до конца или уходит на 50%?
  • Клики на фото vs кнопка "Купить"?
  • "Загрузить ещё" — используется? (FRIC)
  • SOON/Нет в наличии — стопорит? (STOP)

🛍️ Карточка товара (/products/xxx)

  • Смотрит все фото? Или только первое?
  • Читает описание или скипает?
  • Находит кнопку "Купить" сразу? (CTA)
  • Проверяет раздел доставки?
  • Уходит куда — назад в коллекцию или с сайта?

🧭 Навигация / Меню

  • Открывает гамбургер — что ищет? (HIDE)
  • Использует поиск или нет?
  • Петли A→B→A? (LOOP)
  • Категории — понятны ли названия?

🛒 Корзина / Оформление

  • Добавляет в корзину? Или только просматривает?
  • Открывает корзину и закрывает? (DEAD/SPIN)
  • Дошёл до формы адреса?
  • Где последнее действие перед уходом?
📋 Шаг 4 — Быстрый скоринг (1 мин на сессию)
✅ Здоровая сессия
Время >1 мин, линейный маршрут, нашёл товар, добавил в корзину. Ничего делать не нужно.
⚠️ Сессия с трением
Зависал, возвращался, долго искал. Записать: где именно + тип (FRIC/HIDE/LOOP). Если 3+ — гипотеза.
🔴 Сессия-баг
Rage-клик, спиннер без ответа, dead click. Немедленно записать. Если 2+ повтора — фикс P0.

⚙️ Что нужно настроить в Метрике

Рекомендации от 28.05.2026
Метрика сама показала 3 критические рекомендации на странице Сводки.
!
🔴 Цели на кнопки — не настроены
Сейчас Метрика не знает, сколько людей нажали «Купить», «В корзину», «Заказать». Нужно создать цели типа «Клик по кнопке». Это даст данные по CTA-конверсии без кода. Настроить через: Настройки → Цели → + Цель → Клик по кнопке.
!
🔴 Электронная коммерция — события add_to_cart не передаются
Метрика видит покупки, но не видит «Добавление в корзину». Нужно настроить передачу события add через Tilda → Интеграции → Яндекс.Метрика. Без этого не видим где конверсия рвётся.
2
🟡 Воронка checkout — настроить
Создать воронку: главная → /rings или /spring → /cart → /checkout → спасибо. Покажет на каком шаге теряем больше всего. Настройки → Воронки → + Воронка.
3
🟡 Карта скроллинга на /rings и /spring
Открыть Карту скроллинга на самых популярных страницах. Если 70% уходят до середины — CTA кнопка слишком низко. Сдвигаем кнопку «Купить» выше.

Screen Recording (macOS)

CSRF токен из HTML
Работает: fetch('https://metrika.yandex.ru/').then(r=>r.text()).then(h=>{window._csrf=h.match(/csrfToken":"([^"]+)/)[1]}). Токен нужен для POST-запросов. Но i-proxy использует HMAC-секреты (меняются каждую секунду) — обойти невозможно.
▶️
Плеер через get_page_text
РАБОТАЕТ: Открыть /inpage/visor-proto?visit_id=XXX&date=2026-05-28&user_id_hash=YYY&offset=0&id=108324634 → get_page_text → показывает текущую страницу пользователя, устройство, категорию, время (n:nn / n:nn). Нужны только visit_id + user_id_hash из Webvisor списка.
